Post to Textpattern from Textmate With XML-RPC  #

Update 2007/08/08: The XML-RPC posting-from-Textmate-to-Textpattern functionality seems quite buggy and in my opinion is not worth pursuing at this time. One problem I’ve found is the disappearance of the URL title from a Textpattern article when updating a post via Textmate’s Bundles → Blogging → Fetch Post, and as a consequence, a 404 Error.

For those of you who have asked for help with this, I’ve added some clarifications below. Cheers!

- – -

If this post is visible, then this worked. Things that needed to be done:

  1. Download and install Textmate XML-RPC
  2. Apply the fix described in comment #20 here
  3. Make sure you have the Textmate Blogging bundle installed. You can use the GetBundle Bundle to easily get the blogging bundle and any other bundle you might like to bundle into Textmate. Bundle bundle bundle.
  4. Make Textile be recognized as the syntax: Fire up Textmate, then on the menu bar: Bundles → Bundle Editor → Show Bundle Editor → Blogging → Blog Post (Textile), NOT Blog â€” Textile → CLICK ON: untitled.blog.textile, and add Format: 1 under the Title: line like this: 
    Title: ${TM_BLOG_TITLE}
Format: 1
${TM_BLOG_HEADER}
Main entry text
- - - - - -
Main entry continued
  5. Configure the RPC endpoint URL to post to a specific Textpattern section (Menu bar → Bundles → Blogging → Setup Blogs) like this: http://username@site.com/rpc/index.php#sectionname. For example, my site would be set up as follows: 
     Hypertexthero Dlog   http://sgriffee@hypertexthero.com/rpc/index.php#dlog
  6. To post to a site: Menu bar → File → New From Template → Blogging → Blog Post (Textile).
  7. Fill in the Title of the post, the excerpt (main entry text) and body (main entry continued)
  8. Either press Commang-Control-P or go to the Menu bar → Bundles → Blogging → Post To Blog. You will be given a list of blogs you have set up on step v. above. Select one and click OK. You’ll be prompted for your password before being able to post.

MPAA DVD Region Code Greed  #

Subject: DVD
From: Dad

Hi,
Yesterday I had to change the region code to watch the film “2010”. I now get the message (when trying to watch any another film) – ‘The DVD Video disc region code does not match the drive region code. You cannot change the region code for your drive’.
Can’t watch any more films on my iMac!
HELP!!!!!
Luv ya.

Subject: Re: DVD
From: Simon

Hey dad,

You can blame the ignorance and greed of the Motion Picture Association of America for this. They devised the terrible idea of ‘DVD regions’ to essentially try to enforce their own economic model to maximize profit in different areas of the world, and to try to force people to buy their media twice. Apple is also to blame for allowing them to compromise both its hardware and software for such immoral ends.

The good news is that you can view your movies using programs like VLC which bypass all this ‘region code’ foolishness; you can use Region X to reset the devious region-code setting limitations of your DVD drive; and finally, you can learn more and support the Electronic Frontier Foundation’s fight for people’s rights in the digital age (I am a member).

Hope this helps!
Love you,
Simon
