Stop Taking PILs and Lay Head on Pillow and ldconfig, or, Getting Python Imaging Library libjpeg-Related Errors Fixed on RHEL

Friday, July 12 2013

You are using the Python Imaging Library (PIL), which uses libjpeg in your Django project on Red Hat Enterprise Linux (RHEL) and come across the following error in a terminal when trying to install stuff:

"The _imaging C module is not installed."

Or errors like the following in emails when your site breaks:

"TemplateSyntaxError: Couldn't get the thumbnail user_photo/something.jpg: The source file does not appear to be an image."

Here is your solution. And see this for more information. And these may also be helpful:

  • http://stackoverflow.com/a/7528988
  • http://community.webfaction.com/questions/7340/how-to-install-pil-with-truetype-support
  • http://stackoverflow.com/a/3342126
  • Make sure python development package is installed: yum install python-devel
